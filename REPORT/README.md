## Этап 3 – Артефакты сдачи

**Описание:** После разработки вам необходимо будет составить и сдать ряд артефактов в репозиторий GitHub

### Структура репозитория
```
*repo_name*/
 ├── app/
      ├── *код приложения*
      └── README.md
 ├── REPORT.md
 └── *ФИО_дата_тема*.pdf
```

Список артефактов:
- Код приложения
- README.md со следующими пунктами:
  - Список технологий
  - Памятка по запуску
  - Скриншот стартовой страницы
- REPORT.md
  - Суть приложения
  - Абзац про каждую технологию с общим описанием, зачем использовалась, кратким сравнением с аналогами
  - Заключение: рефлексия по проделанной работе, впечатления от технологий.
- Презентация
  - Правила наименования: *ФИО_дата_тема*.pdf
  - Содержание:
    - Суть приложения
    - Список технологий
    - Слайд про каждую технологию с общим описанием, зачем использовалась, кратким сравнением с аналогами
    - Скриншоты работы приложения
    - Заключение

## Критерии оценивания

- Полнота отчета
- Сложность реализации (много технологий с глубоким погружением в каждую – то, к чему надо стремиться)
